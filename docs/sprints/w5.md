# Week 5 — Heatmap, Polish & Social Feed

## Goals

Add the activity heatmap to member profiles, polish the UI for mobile,
and embed the Instagram social feed. This sprint focuses on quality and completeness
over new feature scope.

---

## Deliverables

### Activity Heatmap
- **Heatmap data aggregation API** — aggregates `submissions` + `attendances` per member by day; returns a structured dataset for the past 52 weeks
- **Heatmap component** — GitHub-style contribution graph rendered on the member profile page
  - Each cell = one day; color intensity = number of activities that day
  - Tooltip on hover showing date and activity count

### Member Directory
- **Cohort / campus filter** — members can filter the directory by class year and/or campus (Seneca / York)

### Social Feed
- **Instagram embed** — Elfsight widget embedded on the Home or About page showing recent posts from `@codexperts_seneca` and `@codexperts_yorku`

> Using Elfsight (not Instagram Graph API) — Meta's API requires app review and is too complex for MVP scope.

### Polish
- **Responsive design audit** — review all pages on mobile screen sizes; fix layout breaks, font sizes, spacing
- **UI consistency pass** — ensure colors, spacing, component styles are consistent across all pages

---

## Done When

- [ ] Activity heatmap visible on member profile page with real data
- [ ] Heatmap shows correct color intensity based on submission + attendance activity
- [ ] Member directory can be filtered by cohort and campus
- [ ] Instagram feed embedded and displaying recent posts
- [ ] All pages render correctly on mobile (375px width minimum)
- [ ] No obvious UI inconsistencies across pages
