# Week 3 — Member Features & Coding Problems

## Goals

Build all member-only features and the full coding problem flow end-to-end.
A member should be able to browse problems, write code, run it, and submit — all in the browser.

---

## Deliverables

### Backend
- **FastAPI `/execute` endpoint** — receives code + language, proxies to Piston API, returns stdout/stderr
- **Piston API integration** — supports Python, Java, C++, JavaScript
- **Submission save API** — POST to Supabase `submissions` table (user, problem, code, language, timestamp)

### Member Pages
- **Member directory** — grid/list of all approved members with profile cards (photo, name, school, socials)
  - Filter by cohort, school, status, role
  - Sorted by activity score (submissions + attendance, last 90 days)
- **Member profile page** (`/members/:id`) — view any member's profile
  - Profile header: avatar, name, role/status badges, school, cohort, social links
  - Bio section, activity heatmap (GitHub-style), achievement badges
  - Fields hidden by the member (visibility = off) are not rendered for others
- **Self-edit mode** — shown only when `userId === currentUser.id`, inline on same route
  - Editable fields: photo (upload), bio (300 char), LinkedIn URL, GitHub URL, email
  - Per-field visibility toggle: `Visible to members` / `Hidden` (stored as `profile_visibility` JSONB)
  - Status toggle: Student ↔ Graduate (always visible, no visibility control)
  - Read-only fields: name, school, cohort, role (Admin-managed only)
  - On own profile read view: hidden fields shown with 🔒 icon (only to self)
- **Coding problems list** — all active problems with title, difficulty, deadline
- **Problem detail page** — problem description, constraints, Monaco Editor (VSCode-style in-browser IDE)
  - Language selector (Python, Java, C++, JavaScript)
  - "Run" button → calls `/execute` → shows output below editor
  - "Submit" button → saves code to Supabase
- **View submissions** — see other members' submitted code for each problem
