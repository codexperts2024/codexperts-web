# Week 4 — Admin Panel & QR Attendance

## Goals

Complete the Admin panel and build the QR attendance system end-to-end.
By end of this sprint, an Admin can manage all users and problems,
and members can check in to meetings by scanning a QR code.

---

## Deliverables

### Admin Panel
- **Role management** — Admin can change any member's role (member → executive → admin)
- **Problem CRUD** — Admin/Executive can create, edit, and delete weekly coding problems
- **User management** (from W2) — now includes role change UI alongside approve/reject

### QR Attendance System

**Admin side:**
- **Session creation** — Admin starts an attendance session from the Admin panel; a unique token is generated and stored in the `sessions` table
- **QR code display** — QR code is rendered on screen (encodes the session token)
- **Session management** — Admin can close the session manually; sessions expire automatically after a set time

**Member side:**
- **QR scan UI (mobile)** — member opens the site on their phone, scans the QR code
- The scan triggers a POST to FastAPI `/attendance/verify`

**Backend:**
- **FastAPI `/attendance/verify`** — validates the token, checks the session is still active, records attendance in the `attendances` table
- **Duplicate prevention** — if a member has already checked in to the same session, the request is rejected

---

## Done When

- [ ] Admin can create a QR attendance session — QR code appears on screen
- [ ] Member scans QR on mobile → attendance recorded in Supabase
- [ ] Duplicate scan is blocked with an appropriate error
- [ ] Session can be closed by Admin or expires automatically
- [ ] Admin can change user roles from the panel
- [ ] Admin/Executive can create, edit, and delete problems
