# Week 1 — Foundation & Design

## Goals

Get every team member running the app locally with Google login working.
Lock in the design direction and DB schema before frontend work begins.

> Gary's DB schema and Paul's Figma wireframes are the two hard blockers for Week 2.
> Both must be complete by end of this sprint.

---

## Deliverables

### Infrastructure
- **Next.js 15 project setup** — Tailwind CSS, ESLint, folder structure (`src/app/`, `src/components/`, `src/services/`, etc.)
- **Vercel pipeline** — `develop` branch auto-deploys to preview URL on every push
- **Railway FastAPI** — Hello World endpoint deployed and accessible

### Auth & Database
- **Supabase project** — PostgreSQL schema created (`profiles`, `problems`, `submissions`, `sessions`, `attendances`)
- **Google OAuth** — Supabase Auth configured with Google provider only
- **RLS policies** — Row-Level Security drafted for each table by role
- **Next.js ↔ Supabase session connection** — auth state available client-side

### Frontend
- **Placeholder pages** — all 8 routes scaffolded (`/`, `/about`, `/schedule`, `/events`, `/join`, `/problems`, `/members`, `/admin`)
- **Navbar** — layout scaffolded (role-based slots for items, social icon positions)
- **Social links config** — `src/config/socialLinks.js` with campus-expandable structure
- **Design system** — color palette, font choice, Tailwind base config

### Design
- **Figma wireframes** — rough layout for all 8 pages, approved before Saturday meeting

### Project Management
- **GitHub Projects board** — Kanban set up (Backlog / To Do / In Progress / In Review / Done)
- **All 43 issues created** — labeled, assigned, added to board
