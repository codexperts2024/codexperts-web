# Week 2 — Auth Flow & Public Pages

## Goals

Complete the full authentication flow and build all public-facing pages.
By end of this sprint, a new user should be able to sign up, wait for approval,
and a visitor should be able to browse all public pages.

---

## Deliverables

### Auth & Access Control
- **Pending → approval flow** — new Google sign-in creates a `pending` profile; Admin approves to grant `member` role
- **Admin approval API** — Supabase function to update role from `pending` → `member`
- **Protected Route / Role Guard** — middleware that blocks member-only pages for unauthenticated or pending users
- **Navbar** — role-based visibility (shows/hides items based on auth state), Instagram hover dropdown (public), Discord hover dropdown (members only)

### Public Pages
- **Home** — hero section, club intro, CTAs (Join Us, Problems)
- **About Us** — club description, Executive Board member cards
- **Schedule** — Google Calendar embed showing upcoming meetings and events
- **Events** — event cards with date, title, and description
- **Join Us** — Google Sign-In button, explanation of the approval process

### Admin (partial)
- **Pending user list** — Admin can see who is waiting for approval and approve them

---

## Done When

- [ ] New user signs in with Google → lands on a pending/waiting screen
- [ ] Admin can see the pending list and approve a user
- [ ] Approved user can access member-only pages; pending user cannot
- [ ] All 5 public pages are navigable and visually complete
- [ ] Navbar shows correct items based on auth state
